You can use it as a starting template to make your web appearance and portfolio awesome. It’s based heavily on the magnificent work of Stephen Arsenault, Nathan Randecker and me Sitdisch.

    Local Running

    To enable local operation of this boilerplate on your machine, you have to do the following:

    • open your terminal and define your shell MYSHELL and startup file MYSTARTUPFILE in use
      • e. g. for Zsh 
        MYSHELL="zsh"
MYSTARTUPFILE=".zshrc"
      • or for pure Bash 
        MYSHELL="bash"
MYSTARTUPFILE=".bashrc" # or .bash_profile for macOS
    • install Ruby and Bundler to enable Jekyll
      • Ubuntu
        • update the packages index sudo apt update
        • install Ruby
      sudo apt-get install autoconf ruby-full build-essential zlib1g-dev
        • adapt your startup file
      echo '# Install Ruby Gems to ~/gems' >> ~/$MYSTARTUPFILE
echo 'export GEM_HOME="$HOME/gems"' >> ~/$MYSTARTUPFILE
echo 'export PATH="$HOME/gems/bin:$PATH"' >> ~/$MYSTARTUPFILE
source ~/$MYSTARTUPFILE
        • if your terminal hasn’t closed, proceed with the next step, otherwise, reopen it and redefine MYSHELL and MYSTARTUPFILE
        • install Bundler gem install bundler
      • macOS
        • install the command line tools xcode-select --install
        • install Homebrew
      /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
        • install Ruby brew install ruby
        • adapt your startup file
      echo 'export PATH="/usr/local/opt/ruby/bin:$PATH"' >> ~/$MYSTARTUPFILE
source ~/$MYSTARTUPFILE
        • if your terminal hasn’t closed, proceed with the next step, otherwise, reopen it and redefine MYSHELL and MYSTARTUPFILE
        • install Bundler gem install --user-install bundler
        • execute echo 'export PATH="$HOME/.gem/ruby/X.X.0/bin:$PATH"' >> ~/$MYSTARTUPFILE to adapt your startup file (replace X.X. with your Ruby version ruby -v)
      • Others
    • install nvm and source your startup file again 
      curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.38.0/install.sh | $MYSHELL
source ~/$MYSTARTUPFILE
    • if your terminal has closed, open it again and continue
    • execute nvm install 14.17.0 to install Node.js
    • clone the repository git clone https://github.com/mythemeway/Light-Particle or your fork to your local machine
    • change your current directory to Light-Particle cd Light-Particle
    • run bundle install
    • run npm install

    CONSIDER: If you’ve already installed Ruby, Bundler, nvm and Node.js you only have to do the last four steps.

    That’s it. Now every time you want to run the website locally, go to your Light-Particle directory on your local machine and execute npm start. Your website will then appear at http://localhost:8080.

    If you execute npm run build instead, a one off build of your website will be put into the ~/Light-Particle/docs directory.

    Customization

    Customize your site and user settings e. g. while adapting the scss files under ~/Light-Particle/src/styles/ or the following _config.yml file located at ~/Light-Particle.

    # Site settings
title: # insert a page-name
description: # insert a website description.
baseurl: /Light-Particle

# User settings
username: # insert a username
user_title: # insert your title
user_picture: # insert a path to your pic e.g. "/assets/img/User_picture.png" (size e.g. 250x250)
user_description: # descripe your experiences
email: # insert your_email_address
github_username: # insert your Github username
twitter_username: # insert your Twitter username
youtube_channel_id: # your Youtube channel id
linkedin_username: # insert your Linkedin username
facebook_username: # insert your Facebook username
instagram_username: # insert your Instagram username

# Google Search Console
google_site_verification: # your meta tag (no quotation marks)

    CONSIDER:

    • if you want to use this theme on your main domain e. g. <username>.github.io, just set baseurl: to nothing; otherwise, it will only be used in the specified subdirectory e. g. <username>.github.io/Light-Particle
    • if you set a username e. g. for Twitter the icon will occur automatically

    Setup at GitHub Pages

    Here is one procedure of getting your website published at GitHub Pages:

    1. fork the repository to your GitHub account
    2. publicize your repository on GitHub Pages
    3. clone your fork to your local machine git clone https://github.com/<username>/Light-Particle (replace <username>)
    4. enable local running of your website
    5. customize your website
    6. execute npm run build to put a one off build of your website into the ~/Light-Particle/docs directory.
    7. add git add ., commit git commit -am "my changes" and push git push your changes to GitHub
    8. access your website at https://<username>.github.io/Light-Particle/ (replace <username>) and be proud

    Now every time you want to change your published website, go to your Light-Particle directory on your local machine and repeat steps 5–8.

    CONSIDER:

    • it might take a while that your changes occur at GitHub Pages
    • you may also need to clear the cookies and old site data from your browser
